Caesar Cipher

Julius Caesar used this exact trick 2,000 years ago. Today you'll crack it — and learn how casting, ASCII values, and modular arithmetic make it work in Java.

The Shift Idea

Julius Caesar sent secret messages to his generals by shifting every letter in the alphabet by a fixed number. With a shift of 3: A becomes D, B becomes E — and when you hit the end, it wraps: X becomes A, Y becomes B, Z becomes C.

The shift number is the key. Without it, you can't decode the message.

In Java, we do this mathematically using the ASCII values of characters — every letter is secretly a number.

Java Concepts You'll Use

charAt(i) in a Loop

To encode a whole message, you loop over every character one at a time. charAt(i) grabs the character at position i.

String message = "HELLO";
for (int i = 0; i < message.length(); i++) {
    char c = message.charAt(i);  // grab one character at a time
    // ... do something with c
}

Casting: char ↔ int

Every character has a number behind it — its ASCII value. 'A' is 65, 'B' is 66, ... 'Z' is 90. You can convert between them by casting.

int ascii  = (int) 'A';    // ascii  = 65
char letter = (char) 66;   // letter = 'B'

// The alphabet-position trick: subtract 'A' to get 0–25
int pos = (int)'E' - (int)'A';   // pos = 4  (E is the 5th letter, index 4)

Mod 26 — Wrapping Around

% (mod) gives you the remainder after division. 27 % 26 = 1. This is what makes the alphabet wrap around — just like a clock wraps from 12 back to 1.

Clock analogy: if it's 11 o'clock and you add 3 hours, you get 2 o'clock (not 14). % 26 does the same thing for the alphabet.

// Encoding 'Y' with shift 3:
// Y = position 24
// 24 + 3 = 27
// 27 % 26 = 1   <-- wraps around the alphabet!
// position 1 = 'B'
// So 'Y' --> 'B'

The Full Formula

// Encoding one uppercase letter with a given shift:
int pos     = (int)c - (int)'A';        // step 1: get position 0-25
int shifted = (pos + shift) % 26;       // step 2: shift and wrap
char result = (char)(shifted + 'A');    // step 3: back to a character

// For decoding (shift backwards), the +26 prevents negative numbers:
int shifted = (pos - shift + 26) % 26;

Passing Non-Letters Through

Spaces, punctuation, numbers — we don't touch those. Character.isLetter(c) tells you if a character is a letter.

if (Character.isLetter(c)) {
    // apply the cipher
} else {
    encodedLine += c;  // pass through unchanged
}

Your Task

Write two methods: one to encode a Caesar cipher, one to decode it. Your teacher will provide the ArrayList of lines and the shift value.

// Encodes every letter in every line by the given shift
// Non-letter characters (spaces, punctuation) pass through unchanged
public ArrayList<String> caesarEncode(ArrayList<String> lines, int shift) {
    ArrayList<String> result = new ArrayList<String>();

    for (String line : lines) {
        String encodedLine = "";

        for (int i = 0; i < line.length(); i++) {
            char c = line.charAt(i);   // grab one character at a time

            if (Character.isLetter(c)) {
                // Figure out if it's uppercase or lowercase
                char base = Character.isUpperCase(c) ? 'A' : 'a';

                // Get the alphabet position (0-25)
                int pos = (int)c - (int)base;

                // Shift it and wrap with % 26
                int shifted = (pos + shift) % 26;

                // Convert back to a character and add to the line
                encodedLine += (char)(shifted + (int)base);
            } else {
                encodedLine += c;   // spaces and punctuation stay as-is
            }
        }

        result.add(encodedLine);
    }

    return result;
}

// Decodes by shifting backwards
// The +26 prevents negative numbers when shift > pos
public ArrayList<String> caesarDecode(ArrayList<String> lines, int shift) {
    ArrayList<String> result = new ArrayList<String>();

    for (String line : lines) {
        String decodedLine = "";

        for (int i = 0; i < line.length(); i++) {
            char c = line.charAt(i);

            if (Character.isLetter(c)) {
                char base = Character.isUpperCase(c) ? 'A' : 'a';
                int pos = (int)c - (int)base;
                // TODO: shift backward — use (pos - shift + 26) % 26
                int shifted = (pos - shift + 26) % 26;
                decodedLine += (char)(shifted + (int)base);
            } else {
                decodedLine += c;
            }
        }

        result.add(decodedLine);
    }

    return result;
}
// Gotcha to watch for:

Lowercase letters! 'a' is ASCII 97, not 65. If you only handle uppercase, lowercase letters will come out garbled. The code above uses Character.isUpperCase(c) to pick the right base — make sure you do the same.

For uppercase: base = 'A' (ASCII 65). For lowercase: base = 'a' (ASCII 97). Always subtract and add back the same base.
Test edge cases first. What does encoding 'Z' with shift 1 give? Should be 'A'. What about encoding 'z' with shift 1? Should be 'a'. Try those manually before writing the loop.
For better performance, consider using a StringBuilder instead of encodedLine += c. String concatenation in a loop creates a new object every time; StringBuilder is much faster on long messages.

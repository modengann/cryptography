Caesar Cipher

Julius Caesar used this exact trick 2,000 years ago. Today you'll crack it — and learn how casting, ASCII values, and modular arithmetic make it work in Java.

The Shift Idea

Julius Caesar sent secret messages to his generals by shifting every letter in the alphabet by a fixed number. With a shift of 3: a becomes d, b becomes e — and when you hit the end, it wraps: x becomes a, y becomes b, z becomes c.

The shift number is the key. Without it, you can't decode the message.

In Java, we do this mathematically using the ASCII values of characters — every letter is secretly a number.

Java Concepts You'll Use

charAt(i) in a Loop

To encode a whole message, you loop over every character one at a time. charAt(i) grabs the character at position i.

String message = "hello";
for (int i = 0; i < message.length(); i++) {
    char c = message.charAt(i);  // grab one character at a time
    // ... do something with c
}

Casting: char ↔ int

Every character has a number behind it — its ASCII value. 'a' is 97, 'b' is 98, ... 'z' is 122. You can convert between them by casting.

int ascii   = (int) 'a';   // ascii  = 97
char letter = (char) 98;   // letter = 'b'

// The alphabet-position trick: subtract 'a' to get 0–25
int pos = (int)'e' - (int)'a';   // pos = 4  (e is the 5th letter, index 4)

What Happens If You Just Add?

We'll keep things simple and work with lowercase letters only — your messages should be all lowercase before encoding. Try entering a letter and a shift below to see what happens if you just add without any wrapping.

Enter a lowercase letter and a shift to see what happens.

Since we're using lowercase only, 'a' (97) is always the base. Here's the clean version:

int  pos     = c - 'a';            // how far into the alphabet is this letter? (0–25)
int  shifted = (pos + shift) % 26; // shift it, wrap around if we go past 'z'
char result  = (char)(shifted + 'a'); // turn the position back into a letter

Mod 26 — Wrapping Around

% (mod) gives you the remainder after division. 27 % 26 = 1. This is what makes the alphabet wrap around — just like a clock wraps from 12 back to 1.

Clock analogy: if it's 11 o'clock and you add 3 hours, you get 2 o'clock (not 14). % 26 does the same thing for the alphabet.

// Encoding 'y' with shift 3:
// 'y' - 'a' = position 24
// 24 + 3 = 27
// 27 % 26 = 1   // wraps around!
// position 1 + 'a' = 'b'
// So 'y' --> 'b'

The Full Formula

// Encoding one lowercase letter:
int pos     = c - 'a';                 // step 1: get position 0–25  ('a'=0, 'b'=1, ... 'z'=25)
int shifted = (pos + shift) % 26;      // step 2: shift and wrap back into range
char result = (char)(shifted + 'a');   // step 3: turn the position back into a letter

// Decoding works the same way — just shift backwards.
// The +26 makes sure we never end up with a negative number:
int shifted = (pos - shift + 26) % 26;

Passing Non-Letters Through

Spaces, punctuation, numbers — we don't touch those. Character.isLetter(c) tells you if a character is a letter.

if (Character.isLetter(c)) {
    // apply the cipher
} else {
    encodedLine += c;  // pass through unchanged
}

Handling Upper and Lowercase

Your first version works great for lowercase. But what if the input has a mix — "Hello, World!" instead of "hello, world!"? Java gives you two methods to check:

Character.isUpperCase('A')  // true
Character.isUpperCase('a')  // false
Character.isLowerCase('a')  // true
Character.isLowerCase('A')  // false

You can use these to pick the right base — 'A' (65) for uppercase, 'a' (97) for lowercase — so each letter stays in its own case after the shift.
